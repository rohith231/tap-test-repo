    8.1 Service Models

    Information systems, particularly those based on cloud architecture models, are made up of different service layers. Below are some questions that help the system owner determine if their system is a cloud followed by specific questions to help the system owner determine the type of cloud.
    Question? Answer (Yes/No) Conclusion
    Does the system use virtual machines? {{system_is_cloud.virtual_machine}} A no response means that system is most likely not a cloud.
    Does the system have the ability to expand its capacity to meet customer demand? {{system_is_cloud.expand_capacity}} A no response means that the system is most likely not a cloud.
    Does the system allow the consumer to build anything other than servers? {{system_is_cloud.consumer_build}} A no response means that the system is an IaaS. A yes response means that the system is either a PaaS or a SaaS.
    Does the system offer the ability to create databases? {{system_is_cloud.create_databases}} A yes response means that the system is a PaaS.
    Does the system offer various developer toolkits and APIs? {{system_is_cloud.developer_toolkits}} A yes response means that the system is a PaaS.
    Does the system offer only applications that are available by obtaining a login? {{system_is_cloud.obtaining_login}} A yes response means that system is a SaaS. A no response means that the system is either a PaaS or an IaaS.
